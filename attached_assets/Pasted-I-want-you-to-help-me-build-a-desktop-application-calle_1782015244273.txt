I want you to help me build a desktop application called **MediaVault** (name can be changed later).

The application must be a completely offline, local-first media manager and media player. It should never require cloud services, user accounts, internet connectivity, or online databases. All data must be stored locally on the user's device.

## Primary Purpose

The application should:

* Scan and index local video and audio files.
* Play video and audio files.
* Allow extensive metadata management through custom tagging systems.
* Allow advanced searching and filtering.
* Allow playlist creation and management.
* Handle large media libraries efficiently.

## Platform Requirements

Create this as a desktop application.

Preferred technology stack:

* Frontend: React + TypeScript
* Desktop Framework: Electron or Tauri
* Database: SQLite
* Media Playback: Native video/audio playback libraries
* State Management: Modern React approach

The application should be designed with future scalability in mind.

## External Drive Support

The application must fully support media libraries stored on:

* Internal drives
* External HDDs
* External SSDs
* USB drives
* Network drives (future support)

### Drive Tracking

When scanning media, the application should store:

* Full file path
* File name
* File size
* File creation date
* File modified date
* Media duration
* A file fingerprint/hash
* Drive identifier/volume ID

### Missing Drive Handling

If an external drive is disconnected:

* Do not delete media entries from the database.
* Mark files as "Drive Offline" or "Unavailable".
* Preserve all metadata, playlists, tags, ratings, history, and thumbnails.
* Allow users to see unavailable media if desired through a filter.
* Prevent playback until the drive is reconnected.

### File Relocation Detection

If a file is moved:

* Detect moved files using fingerprints, hashes, metadata, and drive identifiers.
* Automatically relink files when possible.
* Preserve all metadata assignments.
* Preserve playlist membership.
* Preserve playback history.
* Preserve ratings and favorites.

Example:

Old Path:
D:\Videos\Fantasy\Dragon.mp4

New Path:
D:\Media\Fantasy\Dragon.mp4

The application should automatically recognize that this is the same file and update the stored path rather than creating a duplicate entry.

### Library Integrity

The application must never lose metadata simply because:

* A file was renamed.
* A folder was renamed.
* A file was moved.
* An external drive was disconnected.
* A drive letter changed.

The system should prioritize preserving metadata relationships and reconnecting files whenever possible.
